# 📊 FinTech-Data-Analysis
## 專案名稱：數位金融產品申請流程優化分析
### **1. 專案目標**
透過數據分析找出會流失客戶的開戶申請步驟，分析流失原因，並再提出修改方向改善此情況。
<br>

### **2. 使用工具**
* **數據處理：** `MySQL`
使用 Recursive CTE 與 RAND() 函數建立 100 個用戶，模擬開戶申請時客戶流失的現象。
再來利用 COUNT(DISTINCT user_id) 抓取各階段未流失的客戶，確認客戶流失數量的數據，運用漏斗概念顯示各階段用戶流失情形。

* **視覺化分析：** `Power BI`
視覺化呈現客戶流失現象，特別將主要流失客戶階段標註為橘色。後續再加以分析如何解決問題。

### **4. 問題分析**
* **可能原因：**
1. 使用者認為上傳照片繁雜且費時，因此放棄申請。
2. 上傳照片的速度緩慢，使用者因失去耐心而調轉出頁面。
3. 上傳照片容易有反光不符合規格規定，使用者無法排出問題因而跳轉頁面。

<br>

### **5. 改善方法**
1. 優化界面（UIUX）：介面增加上傳照片說明，並加上使用進度條（例如：3秒即可完成），增加使用者等待意願。
2. 優化圖像辨識（前端）：引入 OpenCV.js (WASM) 技術，達到近似 iPhone 掃描偵測機制，使照片可以快速清晰被截取，避免因反光無法上傳照片問題。
3. 提升上傳速度（後端）：優化伺服器併發處理能力，加快上傳速度。

<br>

### **6. 進階追蹤**
後續透過 SQL 擷取更精確的資料，並著重從使用者裝置與年齡兩個面向分析。若數據顯示使用者機型較為老舊，或使用者年齡偏高，則再針對這些使用經驗進行優化。
